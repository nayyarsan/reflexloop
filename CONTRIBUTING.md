# Contributing to reflexloop

reflexloop is designed to be extended. Here's exactly how.

---

## Adding a new agent type

Agents live in `context/agents/<name>/`. The adapter files (`.claude/agents/` and `.github/agents/`) are generated from them.

### 1. Create the context folder

```bash
mkdir -p context/agents/<name>
```

### 2. Write the system prompt

Create `context/agents/<name>/system-prompt.md` with these required sections:

```markdown
## Role
One sentence: what kind of expert is this agent?

## Goal
One sentence: what does a successful session look like?

## Steps
Numbered list of concrete actions the agent takes, in order.
Be specific — vague steps produce vague behaviour.

## Constraints
Bullet list of things the agent must never do.
Each constraint should be a single clear imperative.

## Output Format
Exactly what the agent should emit at the end of every session.
If it's structured data, include the schema.
```

### 3. Write examples

Create `context/agents/<name>/examples.md` with at least one worked example showing:
- The input task
- The agent's plan
- The key outputs
- The summary in the correct Output Format

### 4. Initialise the context files

```bash
touch context/agents/<name>/critiques.jsonl
cat > context/agents/<name>/changelog.md << EOF
# <name> Changelog

## Batch #0 — $(date +%Y-%m-%d)
**Initial version.** <one sentence describing the agent's purpose>
EOF
```

### 5. Create the Claude Code adapter

Create `.claude/agents/<name>.md`:

```markdown
---
name: <name>
description: <one sentence — this is what appears in Claude Code's agent picker>
tools: Read, Write, Edit, Bash, Glob, Grep
---

<!-- SOURCE OF TRUTH: context/agents/<name>/system-prompt.md -->
<!-- Do not edit this file directly. Edit the context file instead. -->

<context_file>context/agents/<name>/system-prompt.md</context_file>
<context_file>context/agents/<name>/examples.md</context_file>
```

### 6. Create the GitHub Copilot adapter

Create `.github/agents/<name>.agent.md`:

```markdown
---
description: <same one sentence>
tools:
  - read_file
  - write_file
  - run_terminal_cmd
  - search_files
model: claude-sonnet-4-6
handoffs:
  - label: "Run critique on this session"
    agent: critique-agent
    prompt: "Critique the session above. Reference context/agents/<name>/system-prompt.md."
    send: false
---

<!-- SOURCE OF TRUTH: context/agents/<name>/system-prompt.md -->

{{{ context/agents/<name>/system-prompt.md }}}

## Examples

{{{ context/agents/<name>/examples.md }}}
```

### 7. Update validate-structure.sh

Add your new agent to `tests/validate-structure.sh`:

```bash
# In the DIRS array, add:
"context/agents/<name>"

# In the check_include block, add:
check_include ".claude/agents/<name>.md" "context/agents/<name>/system-prompt.md"
```

### 8. Run the test suite

```bash
bash tests/validate-structure.sh
bash tests/validate-agent-prompts.sh
```

### 9. Commit

```bash
git add context/agents/<name>/ .claude/agents/<name>.md .github/agents/<name>.agent.md tests/validate-structure.sh
git commit -m "feat(<name>): add new agent type"
```

## Adding a new runtime

reflexloop uses a two-layer model: runtime-agnostic context files + thin adapters per platform.

To add a new runtime (e.g., a different IDE, agent host, or CI system):

### 1. Create an adapter directory

```bash
mkdir -p .<runtime>/agents
mkdir -p .<runtime>/hooks
```

### 2. Write adapter files

Each adapter must:
- Load from `context/agents/<name>/system-prompt.md`
- Load from `context/agents/<name>/examples.md`
- Reference the context files rather than duplicating content

### 3. Wire the post-session hook

Your hook must call:
```bash
bash scripts/run-critique.sh <session-id> <agent-name>
```

The session ID must be a unique string per session — any identifier the runtime provides will do.

### 4. Document the hook event

Update `README.md`'s runtime integration table with:
- What event fires after a session ends
- How the session ID is exposed
- Any runtime-specific quirks

## Enterprise templates

The `context/agents/` folder includes two pre-tuned templates:

- `dev-agent-regulated/` — for regulated environments (finance, healthcare): adds PII constraints, compliance notes, and audit trail requirements to the output format
- `dev-agent-platform/` — for internal platform teams: adds downstream impact awareness, migration path requirements, and rollback planning

To use a template, copy its folder and rename it:
```bash
cp -r context/agents/dev-agent-regulated context/agents/my-team-agent
# Then edit context/agents/my-team-agent/system-prompt.md
```

## Code style

- Shell scripts: `set -euo pipefail`, use `python` (not `python3`) for JSON processing
- Use `trap cleanup EXIT` in any test that modifies shared state
- Keep system prompts under 2000 characters where possible (run `bash tests/test-safety-checks.sh` to check)
- Every new agent needs a `## Role`, `## Goal`, `## Steps`, and `## Output Format` section

## Pull requests

Open PRs against `master`. Include:
- What agent/runtime/template was added or changed
- Output of `bash tests/validate-structure.sh && bash tests/validate-agent-prompts.sh`
- For new agents: at least one example in `examples.md`
